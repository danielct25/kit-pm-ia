> Este canvas é projetado para ajudar a identificar e avaliar oportunidades de negócio que podem ser destravadas com Inteligência Artificial. É uma ferramenta de brainstorming e análise estratégica para PMs e líderes de produto.

---

### 1. Problema & Proposta de Valor

| Seção | Descrição |
| :--- | :--- |
| **Problema/Necessidade** | Qual problema do cliente ou necessidade de negócio estamos abordando? (Derivado do Problem Statement) |
| **Proposta de Valor Única** | Como a IA nos permite criar uma proposta de valor que é 10x melhor que as alternativas existentes? |
| **Métricas de Sucesso (Negócio)** | Quais são os KPIs de negócio que esperamos impactar? (Ex: Aumento de receita, redução de custos, aumento de engajamento) |

### 2. Capacidades da IA & Dados

| Seção | Descrição |
| :--- | :--- |
| **Capacidade de IA** | Qual é a principal capacidade de IA que será usada? (Ex: Classificação, Regressão, Geração, Detecção de Anomalia) |
| **Input do Modelo (Entrada)** | Que dados o modelo de IA precisará para fazer uma predição ou gerar um resultado? |
| **Output do Modelo (Saída)** | Qual é o resultado que o modelo de IA irá gerar? (Ex: Uma categoria, um número, um texto, uma imagem) |
| **Fonte de Dados (Data Pipeline)** | De onde virão os dados? Como serão coletados, limpos e preparados para o modelo? |
| **Vantagem Competitiva de Dados** | Temos acesso a dados únicos ou proprietários que criam uma barreira de entrada para competidores? |

### 3. Experiência do Usuário & Workflow

| Seção | Descrição |
| :--- | :--- |
| **Jornada do Usuário (com IA)** | Como a IA se encaixa na jornada do usuário? Onde e como o usuário interage com a capacidade da IA? |
| **Confiança e Explicabilidade (XAI)** | Como vamos construir a confiança do usuário no resultado da IA? Precisamos explicar como o modelo chegou àquela conclusão? |
| **Tratamento de Erros do Modelo** | O que acontece quando a IA erra ou tem baixa confiança? Qual é o plano B ou o fallback para o usuário? (Ex: Sugerir ação manual, pedir feedback) |
| **Feedback Loop** | Como podemos usar as interações do usuário e os resultados (bons ou ruins) para retroalimentar e melhorar o modelo de IA continuamente? |

### 4. Viabilidade & Riscos

| Seção | Descrição |
| :--- | :--- |
| **Complexidade Técnica** | Quão difícil é construir, treinar e implantar este modelo de IA? (Use uma escala: Baixa, Média, Alta) |
| **Recursos Necessários** | Que time e habilidades são necessários? (Ex: Engenheiro de ML, Cientista de Dados, Especialista de Domínio) |
| **Riscos Éticos e de Segurança** | Quais são os principais riscos de privacidade, segurança, viés e conformidade regulatória? |
| **Estratégia de Go-to-Market** | Como vamos lançar essa capacidade? (Ex: Beta fechado, A/B testing, lançamento para um segmento específico) |

### Exemplo Simplificado (Recomendação de Filmes)

*   **Problema:** Usuários gastam muito tempo decidindo o que assistir.
*   **Proposta de Valor:** Recomendações personalizadas que acertam em cheio no seu gosto, economizando seu tempo.
*   **Capacidade de IA:** Filtragem Colaborativa (um tipo de sistema de recomendação).
*   **Input do Modelo:** Histórico de filmes assistidos e avaliados pelo usuário e por usuários similares.
*   **Output do Modelo:** Uma lista ordenada de filmes que o usuário provavelmente gostará.
*   **Jornada do Usuário:** A lista de recomendações aparece na tela inicial do aplicativo.
*   **Tratamento de Erros:** Se o usuário não gostar de uma recomendação, ele pode marcar "Não tenho interesse" para refinar o modelo.
*   **Feedback Loop:** As avaliações e o comportamento de assistir (ou não) aos filmes recomendados são usados para retreinar o modelo.
